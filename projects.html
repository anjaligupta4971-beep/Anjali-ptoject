HTML Fundamentals with Examples

Learn HTML basics with practical examples and explanations

1. HTML Document Structure

Every HTML document needs a basic structure. The <!DOCTYPE html> declaration tells the browser this is an HTML5 document. The <html> tag is the root element, <head> contains metadata, and <body> contains visible content.

Basic HTML Structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is my first paragraph.</p>
</body>
</html>
Key Points:
  • <!DOCTYPE html> - HTML5 document type declaration
  • <html lang="en"> - Root element with language attribute
  • <head> - Contains metadata (not visible on page)
  • <meta charset="UTF-8"> - Character encoding
  • <title> - Page title (shown in browser tab)
  • <body> - Contains all visible content

2. HTML Headings and Text Formatting

HTML provides six levels of headings (h1-h6) and various text formatting tags. Headings create a hierarchy, with h1 being the most important.

Headings and Text Formatting:

<h1>Main Heading (Largest)</h1>
<h2>Subheading</h2>
<h3>Section Heading</h3>
<h4>Subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

<p>This is a paragraph. It contains regular text.</p>

<p>
    <strong>This text is bold</strong> and 
    <em>this text is italic</em>. 
    <u>This text is underlined</u> and 
    <mark>this text is highlighted</mark>.
</p>

<p>
    <small>Small text</small> and 
    <del>deleted text</del> and 
    <ins>inserted text</ins>.
</p>
Text Formatting Tags:
  • <strong> - Bold text (semantic importance)
  • <em> - Italic text (emphasis)
  • <u> - Underlined text
  • <mark> - Highlighted text
  • <small> - Smaller text
  • <del> - Deleted text (strikethrough)
  • <ins> - Inserted text (underline)

3. HTML Lists (Ordered and Unordered)

Lists help organize information. Use <ul> for unordered lists (bullets) and <ol> for ordered lists (numbers).

Creating Lists:

<!-- Unordered List (Bullet Points) -->
<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

<!-- Ordered List (Numbered) -->
<ol>
    <li>Step one</li>
    <li>Step two</li>
    <li>Step three</li>
</ol>

<!-- Nested List -->
<ul>
    <li>Frontend
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JavaScript</li>
        </ul>
    </li>
    <li>Backend
        <ul>
            <li>Node.js</li>
            <li>Python</li>
        </ul>
    </li>
</ul>
List Types:
  • <ul> - Unordered list (bullet points)
  • <ol> - Ordered list (numbers/letters)
  • <li> - List item (used in both types)
  • Lists can be nested inside each other

4. HTML Links and Images

Links connect pages together, and images add visual content. The <a> tag creates links, and <img> displays images.

Links and Images:

<!-- External Link -->
<a href="https://www.example.com" target="_blank">
    Visit Example Website
</a>

<!-- Internal Link -->
<a href="about.html">About Page</a>

<!-- Link to Email -->
<a href="mailto:email@example.com">Send Email</a>

<!-- Image -->
<img src="image.jpg" alt="Description of image" width="300" height="200">

<!-- Image as Link -->
<a href="page.html">
    <img src="thumbnail.jpg" alt="Clickable image">
</a>

<!-- Link with Image and Text -->
<a href="https://github.com">
    <img src="github-icon.png" alt="GitHub">
    Visit GitHub
</a>
Attributes Explained:
  • href - URL or file path for links
  • target="_blank" - Opens link in new tab
  • src - Image source path
  • alt - Alternative text (important for accessibility)
  • width and height - Image dimensions

5. HTML Forms

Forms collect user input. The <form> element wraps form controls like text inputs, buttons, checkboxes, and dropdowns.

Complete Form Example:

<form action="/submit" method="POST">
    <!-- Text Input -->
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required placeholder="Enter your name">
    
    <!-- Email Input -->
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <!-- Password Input -->
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>
    
    <!-- Textarea -->
    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4" cols="50"></textarea>
    
    <!-- Dropdown Select -->
    <label for="country">Country:</label>
    <select id="country" name="country">
        <option value="usa">United States</option>
        <option value="uk">United Kingdom</option>
        <option value="canada">Canada</option>
    </select>
    
    <!-- Radio Buttons -->
    <fieldset>
        <legend>Gender:</legend>
        <input type="radio" id="male" name="gender" value="male">
        <label for="male">Male</label>
        <input type="radio" id="female" name="gender" value="female">
        <label for="female">Female</label>
    </fieldset>
    
    <!-- Checkbox -->
    <input type="checkbox" id="newsletter" name="newsletter">
    <label for="newsletter">Subscribe to newsletter</label>
    
    <!-- Submit Button -->
    <button type="submit">Submit Form</button>
    <button type="reset">Reset</button>
</form>
Form Elements:
  • <form> - Container for form elements
  • action - Where form data is sent
  • method - HTTP method (GET or POST)
  • <label> - Describes form inputs
  • <input> - Various input types (text, email, password, etc.)
  • required - Makes field mandatory
  • placeholder - Hint text in input field

6. HTML Semantic Elements

Semantic HTML5 elements give meaning to content structure. They help with SEO and accessibility by clearly defining page sections.

Semantic Layout:

<header>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>
            <li><a href="about.html">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <header>
            <h1>Article Title</h1>
            <p>Published on <time datetime="2024-01-15">January 15, 2024</time></p>
        </header>
        
        <section>
            <h2>Introduction</h2>
            <p>Article content goes here...</p>
        </section>
        
        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
            </ul>
        </aside>
    </article>
</main>

<footer>
    <p>© 2024 My Website. All rights reserved.</p>
</footer>
Semantic Elements:
  • <header> - Page or section header
  • <nav> - Navigation links
  • <main> - Main content area
  • <article> - Independent content piece
  • <section> - Thematic grouping
  • <aside> - Sidebar content
  • <footer> - Page footer
  • <time> - Date/time information

7. HTML Tables

Tables organize data into rows and columns. Use them for displaying structured information like schedules, data lists, or comparisons.

Table Structure:

<table border="1">
    <caption>Student Grades</caption>
    <thead>
        <tr>
            <th>Name</th>
            <th>Subject</th>
            <th>Grade</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Doe</td>
            <td>Mathematics</td>
            <td>A</td>
        </tr>
        <tr>
            <td>Jane Smith</td>
            <td>Science</td>
            <td>B+</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="3">Total Students: 2</td>
        </tr>
    </tfoot>
</table>
Table Elements:
  • <table> - Container for table
  • <caption> - Table title/description
  • <thead> - Table header section
  • <tbody> - Table body (main data)
  • <tfoot> - Table footer
  • <tr> - Table row
  • <th> - Header cell (bold by default)
  • <td> - Data cell
  • colspan - Merge columns

Project 1: Interactive Calculator

A fully functional calculator built with HTML, CSS, and JavaScript. Features include basic arithmetic operations and a clean, modern UI.

HTML Structure:

<div class="calculator">
    <div class="display">
        <input type="text" id="result" readonly>
    </div>
    <div class="buttons">
        <button onclick="clearDisplay()">C</button>
        <button onclick="appendNumber('7')">7</button>
        <button onclick="appendNumber('8')">8</button>
        <button onclick="appendNumber('9')">9</button>
        <button onclick="appendOperator('+')">+</button>
        <!-- More buttons -->
    </div>
</div>

JavaScript Logic:

let display = document.getElementById('result');
let currentInput = '';

function appendNumber(num) {
    currentInput += num;
    display.value = currentInput;
}

function appendOperator(op) {
    if (currentInput !== '' && !isNaN(currentInput[currentInput.length - 1])) {
        currentInput += op;
        display.value = currentInput;
    }
}

function calculate() {
    try {
        currentInput = eval(currentInput).toString();
        display.value = currentInput;
    } catch (error) {
        display.value = 'Error';
        currentInput = '';
    }
}

function clearDisplay() {
    currentInput = '';
    display.value = '';
}

Project 2: Todo List Application

A task management application with add, edit, delete, and mark-as-complete functionality. Uses local storage to persist data.

HTML Structure:

<div class="todo-container">
    <h1>My Todo List</h1>
    <div class="input-section">
        <input type="text" id="todoInput" placeholder="Add a new task...">
        <button onclick="addTodo()">Add</button>
    </div>
    <ul id="todoList"></ul>
</div>

JavaScript Implementation:

let todos = JSON.parse(localStorage.getItem('todos')) || [];

function addTodo() {
    const input = document.getElementById('todoInput');
    const text = input.value.trim();
    
    if (text !== '') {
        todos.push({
            id: Date.now(),
            text: text,
            completed: false
        });
        input.value = '';
        saveTodos();
        renderTodos();
    }
}

function toggleTodo(id) {
    todos = todos.map(todo =>
        todo.id === id ? { ...todo, completed: !todo.completed } : todo
    );
    saveTodos();
    renderTodos();
}

function deleteTodo(id) {
    todos = todos.filter(todo => todo.id !== id);
    saveTodos();
    renderTodos();
}

function saveTodos() {
    localStorage.setItem('todos', JSON.stringify(todos));
}

function renderTodos() {
    const list = document.getElementById('todoList');
    list.innerHTML = todos.map(todo => `
        <li class="${todo.completed ? 'completed' : ''}">
            <span onclick="toggleTodo(${todo.id})">${todo.text}</span>
            <button onclick="deleteTodo(${todo.id})">Delete</button>
        </li>
    `).join('');
}

renderTodos();

Project 4: Form Validation

A contact form with real-time validation using JavaScript. Includes email format checking, required field validation, and user feedback.

HTML Form:

<form id="contactForm" onsubmit="validateForm(event)">
    <div class="form-group">
        <label for="name">Name:</label>
        <input type="text" id="name" required>
        <span class="error" id="nameError"></span>
    </div>
    
    <div class="form-group">
        <label for="email">Email:</label>
        <input type="email" id="email" required>
        <span class="error" id="emailError"></span>
    </div>
    
    <div class="form-group">
        <label for="message">Message:</label>
        <textarea id="message" rows="5" required></textarea>
        <span class="error" id="messageError"></span>
    </div>
    
    <button type="submit">Submit</button>
</form>

JavaScript Validation:

function validateForm(event) {
    event.preventDefault();
    
    const name = document.getElementById('name').value.trim();
    const email = document.getElementById('email').value.trim();
    const message = document.getElementById('message').value.trim();
    
    let isValid = true;
    
    // Name validation
    if (name.length < 3) {
        showError('nameError', 'Name must be at least 3 characters');
        isValid = false;
    } else {
        clearError('nameError');
    }
    
    // Email validation
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(email)) {
        showError('emailError', 'Please enter a valid email address');
        isValid = false;
    } else {
        clearError('emailError');
    }
    
    // Message validation
    if (message.length < 10) {
        showError('messageError', 'Message must be at least 10 characters');
        isValid = false;
    } else {
        clearError('messageError');
    }
    
    if (isValid) {
        alert('Form submitted successfully!');
        document.getElementById('contactForm').reset();
    }
}

function showError(elementId, message) {
    document.getElementById(elementId).textContent = message;
}

function clearError(elementId) {
    document.getElementById(elementId).textContent = '';
}
